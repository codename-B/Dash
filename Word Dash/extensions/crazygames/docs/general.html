General

This module provides general purpose CrazyGames SDK functions.

Note

Make sure to call crazy_init before stating to use the extension.

Functions

The following module functions are available:

  • crazy_init
  • crazy_get_environment
  • crazy_started


Back To Top

crazy_init

This function initializes the CrazyGames SDK for use within a GameMaker HTML5 export. This must be called before using any other CrazyGames functions.


Syntax:

crazy_init(callback_success, callback_failed)
Argument Type Description
callback_success Function Function to execute if initialization succeeds.
callback_failed Function Function to execute if initialization fails, receiving the error as a parameter.




Returns:

N/A


Example:

crazy_init(
    function() { show_debug_message("CrazyGames SDK initialized successfully!"); },
    function(error) { show_debug_message("Failed to initialize CrazyGames SDK: " + string(error)); }
);

crazy_get_environment

This function retrieves the current environment information from the CrazyGames SDK. This may include values such as "local" or "crazygames".


Syntax:

crazy_get_environment()




Returns:

String


Example:

if (env != undefined) {
    show_debug_message("Running in CrazyGames environment: " + env);
} else {
    show_debug_message("CrazyGames SDK not initialized.");
}

crazy_started

This function checks whether the CrazyGames SDK has been successfully initialized.


Syntax:

crazy_started()




Returns:

Boolean


Example:

if (crazy_started()) {
    show_debug_message("CrazyGames SDK is ready!");
} else {
    show_debug_message("CrazyGames SDK is not initialized yet.");
}
